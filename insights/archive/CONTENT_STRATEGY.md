## タイトル戦略

### 成功パターン（高PV記事）

**構造**:
```
[具体的な技術/課題] + [問題提起/解決策] + [数字や実例]
```

**成功例の分析**:

**1. KakeiBon記事（1,088 views）**:
```
タイトル:
  "KakeiBon By Rust"は53歳(もうじき54)によって開発されました

要素:
  ✅ 技術: Rust
  ✅ 実例: KakeiBon
  ✅ 数字: 53歳
  ✅ 検索キーワード: Rust, 開発, 実例, 53歳

なぜ伸びた:
  - AI+Rustの実例を探している人にヒット
  - 年齢がストーリー性を加える
  - 具体的なプロジェクト名で検索される
```

**2. プロンプトエンジニアリング記事（562 views）**:
```
タイトル:
  【その１】プロンプトエンジニアリングの常識が成果を妨げている理由

要素:
  ✅ 技術: プロンプトエンジニアリング
  ✅ 問題提起: 常識が成果を妨げる
  ✅ 検索キーワード: プロンプトエンジニアリング, 成果, 課題

なぜ伸びた:
  - 具体的な課題を持つ人にヒット
  - 「常識が妨げる」という逆説的表現
  - 実務の悩みに直結
```

**3. システム開発シリーズ（142-562 views）**:
```
タイトル:
  システム開発がなぜうまくいかない？を解決するひとつの方法

要素:
  ✅ 課題: システム開発がうまくいかない
  ✅ 解決策: 解決するひとつの方法
  ✅ 検索キーワード: システム開発, うまくいかない, 解決

なぜ伸びた（検索1位）:
  - 検索意図と完全一致
  - 具体的な問題提起
  - 解決策を提示
```

### 失敗パターン（低PV記事）

**おっちゃん記事（210 views）**:
```
タイトル:
  『53歳のおっちゃん、新しいプロジェクトを立ち上げる』の巻

問題点:
  ❌ 何のプロジェクトか不明
  ❌ 技術キーワードがない
  ❌ 検索されにくい表現
  ❌ エモーショナルすぎる

検索されないキーワード:
  - 「おっちゃん プロジェクト」
  - 「53歳 立ち上げる」

なぜ伸びなかった:
  - 実務層の検索キーワードとミスマッチ
  - タイトルから内容が分からない
  - Qiita内部の推薦のみ（検索流入が少ない）
```

## タイトル作成のチェックリスト

### 必須要素

**1. 検索キーワードが明確か？**
```
質問:
  読者は何と検索してこの記事に辿り着くか？

確認:
  そのキーワードがタイトルに入っているか？

例:
  ✅ 「Rust AI 実例」→ "KakeiBon By Rust"
  ✅ 「システム開発 うまくいかない」→ "システム開発がなぜうまくいかない？"
  ❌ 「おっちゃん プロジェクト」→ 検索されない
```

**2. 記事の内容が一目で分かるか？**
```
質問:
  タイトルだけで「何について書いてあるか」が分かるか？

確認:
  技術名、課題、解決策が明確か？

例:
  ✅ "プロンプトエンジニアリングの常識が成果を妨げている理由"
     → プロンプトエンジニアリングの課題について
  ❌ "53歳のおっちゃん、新しいプロジェクトを立ち上げる"
     → 何のプロジェクト？内容不明
```

**3. ターゲット読者に響くか？**
```
質問:
  企業の技術評価チーム、実務の開発者にとって価値があると分かるか？

確認:
  実用的な情報だと伝わるか？

例:
  ✅ 「システム開発がなぜうまくいかない？を解決する」
     → 実務の課題解決
  ❌ 「おっちゃん、新しいプロジェクトを立ち上げる」
     → エモーショナル、実務性不明
```

**4. 数字や実例が入っているか？**
```
質問:
  具体性があるか？

確認:
  数字、実例、プロジェクト名が入っているか？

例:
  ✅ "53歳"、"KakeiBon"、"35,000行"、"2人月"
  ✅ "100%成功"、"生産性15-30倍"
  ❌ "新しいプロジェクト"（曖昧）
```

## コンテンツの質的特徴

### 高PV記事の共通点

**具体性**:
```
✅ 技術名が明確（Rust, Tauri, Blockly）
✅ 数字が具体的（35,000行, 2人月, 525テスト）
✅ 実例がある（KakeiBon, Promps）
```

**実務性**:
```
✅ 実際の課題を扱う（システム開発がうまくいかない）
✅ 解決策を提示（こうすれば解決する）
✅ 実証実績がある（テスト100%成功）
```

**検索適合性**:
```
✅ 検索キーワードとタイトルが一致
✅ 検索意図と内容が一致
✅ Google検索1位（複数記事）
```

### 低PV記事の共通点

**曖昧性**:
```
❌ 何の記事か不明（おっちゃん、新しいプロジェクト）
❌ 技術キーワードがない
❌ 検索されにくい表現
```

**エモーショナル偏重**:
```
❌ ストーリー性重視
❌ 実務性が不明確
❌ 主流読者（実務層）とミスマッチ
```
